# User Preferences

## Documentation and Summaries

- ❌ **DO NOT** create files with task summaries or work recaps
- ❌ **DO NOT** create files like:
  - `SUMMARY.md`
  - `CHANGES_SUMMARY.md`
  - `WORK_DONE.md`
  - `TASK_REPORT.md`
  - Or any similar summary/report files
- ✅ **DO** update existing documentation files when relevant (README.md, CHANGELOG.md, docs/)
- ✅ **DO** provide a brief verbal summary in the chat response

## Rationale

Summary files are:

- Redundant (information already in git history and CHANGELOG.md)
- Create noise in the repository
- Require maintenance and cleanup
- Not part of the project's documentation structure

## What to Do Instead

When completing work:

1. Update CHANGELOG.md with changes (following changelog.md guidelines)
2. Update relevant documentation files if needed
3. Provide a concise verbal summary in the chat
4. Let git commit messages and history serve as the detailed record
